👮 Compliance Officer Guide

Complete guide for compliance officers managing ERC-3643 compliance

⚠️ Restricted Access

This guide is for authorized Compliance Officers only. Your account must have COMPLIANCE_OFFICER role.

🎯 Role Overview

As a Compliance Officer, you are responsible for:

  • ✅ Verifying investor identities
  • ✅ Monitoring transfers for compliance
  • ✅ Managing blocked addresses
  • ✅ Enforcing jurisdiction restrictions
  • ✅ Setting investment limits
  • ✅ Generating compliance reports

🔐 Accessing Compliance Dashboard

Prerequisites

  • Compliance Officer Role: Your account must have COMPLIANCE_OFFICER role
  • Wallet Connected: Connect your MetaMask wallet
  • Identity Verified: Your own identity must be VERIFIED

Login Steps

  1. Visit Ujamaa DeFi Platform
  2. Click "Login"
  3. Enter your credentials:
    • Email: compliance@ujamaa-defi.com
    • Password: [Your password]
  4. Connect your wallet
  5. Navigate to "Compliance Dashboard"

📋 Identity Verification Workflow

Review Pending Identities

  1. Go to "Compliance" → "Identity Review"
  2. View pending applications:
    • Investor name
    • Jurisdiction
    • Investor type
    • Submission date
    • Time remaining (24h window)

Verify an Identity

For APPROVED applications:

  1. Click "Review" on the application
  2. Review submitted documents:
    • ✅ ID document (passport, national ID)
    • ✅ Proof of address
    • ✅ Additional documents (if applicable)
  3. Verify jurisdiction is allowed (not on blocked list)
  4. Click "Approve"
  5. Transaction will be submitted to blockchain
  6. Confirm transaction in MetaMask
  7. Identity status changes to VERIFIED

For REJECTED applications:

  1. Click "Review" on the application
  2. Click "Reject"
  3. Provide rejection reason:
    • Invalid documents
    • Blocked jurisdiction
    • Incomplete information
    • Other (specify)
  4. Click "Submit Rejection"
  5. Identity status changes to REJECTED

🚫 Managing Blocked Addresses

Block an Address

For sanctioned or fraudulent addresses:

  1. Go to "Compliance" → "Blocked Addresses"
  2. Click "Block Address"
  3. Enter address to block
  4. Select blocking reason:
    • OFAC sanctions
    • UN sanctions
    • EU sanctions
    • Fraud investigation
    • Other compliance reason
  5. Add notes (optional but recommended)
  6. Click "Block"
  7. Confirm transaction in MetaMask
  8. Address is now blocked from all transfers

Unblock an Address

If address was blocked in error:

  1. Go to "Compliance" → "Blocked Addresses"
  2. Find the blocked address
  3. Click "Unblock"
  4. Provide unblocking reason
  5. Submit transaction
  6. Address can now transfer tokens

💰 Setting Investment Limits

Set Individual Limit

For specific investors:

  1. Go to "Compliance" → "Investment Limits"
  2. Search for investor by address or name
  3. Click "Set Limit"
  4. Enter new limit (in wei, 18 decimals):
    • Example: €50,000 = 50000000000000000000000
  5. Click "Update"
  6. Confirm transaction
  7. Limit is now enforced for that investor

Set Global Limits

For all investors:

  1. Go to "Compliance" → "Investment Limits"
  2. Click "Global Settings"
  3. Update global limits:
    • Minimum investment (default: €1,000)
    • Maximum retail investment (default: €50,000)
    • Maximum institutional investment (default: €10,000,000)
  4. Click "Update Global Limits"
  5. Confirm transaction
  6. New limits apply to all future investments

📊 Compliance Reports

Generate Daily Report

  1. Go to "Compliance" → "Reports"
  2. Select "Daily Activity Report"
  3. Choose date
  4. Click "Generate"
  5. Report includes:
    • Identities verified
    • Identities rejected
    • Addresses blocked
    • Transfers blocked
    • Jurisdiction changes
  6. Download as PDF or CSV

Generate Transfer Report

  1. Go to "Compliance" → "Reports"
  2. Select "Transfer Compliance Report"
  3. Choose date range
  4. Click "Generate"
  5. Report includes:
    • All transfers attempted
    • Successful transfers
    • Failed transfers
    • Failure reasons
    • Compliance checks performed
  6. Download as PDF or CSV

⚠️ Compliance Best Practices

Identity Verification

  • ✅ Verify documents carefully - Check for authenticity
  • ✅ Check jurisdiction - Ensure not on blocked list
  • ✅ Document decisions - Add notes for all actions
  • ✅ 24-hour window - Review within 24 hours of submission
  • ✅ Escalate suspicious - Report suspicious applications

Transfer Monitoring

  • ✅ Review blocked transfers - Check why transfers failed
  • ✅ Monitor large transfers - Review transfers >€100,000
  • ✅ Check patterns - Look for suspicious patterns
  • ✅ Update blocked list - Add new sanctioned addresses promptly

Record Keeping

  • ✅ Document all actions - Add notes for all decisions
  • ✅ Generate reports - Keep daily/weekly reports
  • ✅ Maintain audit trail - All actions are on-chain
  • ✅ Backup data - Download reports regularly

🔐 Security Guidelines

Protect Your Account

  • 🔒 Use strong password - Minimum 12 characters
  • 🔒 Enable 2FA - If available
  • 🔒 Protect private key - Never share your keys
  • 🔒 Use dedicated wallet - Separate from personal wallet
  • 🔒 Log out when done - Don't leave session open

Authorized Actions Only

  • ⚠️ Don't abuse power - Only block for valid reasons
  • ⚠️ Don't share credentials - Keep your account secure
  • ⚠️ Follow procedures - Use established workflows
  • ⚠️ Report issues - Escalate suspicious activity
