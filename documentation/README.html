📄📊 Documentation Standards

Naming Convention

All files follow the pattern: XX_DESCRIPTION_NAME.md
- XX = Two-digit number (01-99)
- DESCRIPTION_NAME = Uppercase with underscores

Version Control

  • Major versions: X.0 (breaking changes)
  • Minor versions: X.Y (updates, clarifications)
  • Review cycle: Quarterly or after material changes

Cross-References

  • All internal links use relative paths
  • Folder README files link to parent index
  • Main index links to all folder indexes

📄✅ Success Criteria

Documentation is complete when:
- ✅ All files follow XX_UPPERCASE_DESCRIPTION convention
- ✅ All internal links are functional
- ✅ Numbering is consistent and gap-free
- ✅ READMEs reflect the current state
- ✅ Cross-references are accurate
