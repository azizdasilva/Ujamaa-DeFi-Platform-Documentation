♿ Accessibility Guidelines

Making UJAMAA accessible to everyone

📖 Document Info

Status: ✅ Complete | Audience: All Team Members

📑 Our Commitment

UJAMAA DeFi Platform is committed to ensuring digital accessibility for people with disabilities. We strive to meet WCAG 2.1 AA standards.

✅ Accessibility Standards

WCAG 2.1 AA Compliance
  • Perceivable - Information must be presentable to users
  • Operable - Interface must be operable by all
  • Understandable - Information must be understandable
  • Robust - Content must be robust enough for various technologies

🎨 Visual Accessibility

RequirementStandard
Color Contrast4.5:1 minimum ratio
Text SizeScalable to 200%
Color UsageNot sole means of conveying info
Focus IndicatorsVisible for keyboard navigation

⌨️ Keyboard Navigation

  • All functionality keyboard accessible
  • Logical tab order
  • Skip navigation links
  • No keyboard traps
  • Visible focus indicators

📱 Screen Reader Support

  • Proper heading hierarchy
  • Alt text for images
  • ARIA labels where needed
  • Descriptive link text
  • Form labels associated

📝 Content Guidelines

Writing Accessibly
  • Use clear, simple language
  • Define acronyms
  • Use descriptive headings
  • Provide text alternatives
  • Avoid jargon

🧪 Testing

  • Automated accessibility testing
  • Manual keyboard testing
  • Screen reader testing
  • User testing with disabilities
  • Regular audits

📞 Feedback

  • Report Issues: accessibility@ujamaa.defi
  • Response Time: Within 5 business days
  • Accommodations: Available upon request
