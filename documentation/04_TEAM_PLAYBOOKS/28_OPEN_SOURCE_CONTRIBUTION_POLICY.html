🔓 Open Source Contribution Policy

Guidelines for open source contributions

📖 Document Info

Status: ✅ Complete | Audience: Developers

📑 Our Commitment

Ujamaa DeFi Platform believes in giving back to the open source community. This policy outlines how team members can contribute to open source projects.

✅ Supported Activities

What We Support
  • Contributing to dependencies we use
  • Fixing bugs in open source projects
  • Adding features to relevant projects
  • Writing documentation
  • Maintaining our own open source projects

⏰ Time Allocation

ActivityTime Allowed
Work Hours10% of work time
HackathonsCompany-sponsored events
MaintenanceAs needed for our projects
Personal ProjectsPersonal time only

📝 Approval Process

  1. Identify contribution opportunity
  2. Submit open source request form
  3. Explain relevance to work
  4. Manager approval
  5. Legal review (if needed)
  6. Make contribution

⚖️ IP Considerations

  • Company-owned code requires approval
  • Personal projects on personal time are yours
  • Don't use company resources for personal projects
  • Follow license requirements

🏆 Our Open Source Projects

  • Ujamaa SDKs and libraries
  • Smart contract libraries
  • Developer tools
  • Documentation and guides

📞 Questions

  • Engineering: engineering@ujamaa.io
  • Legal: legal@ujamaa.io
  • Open Source: opensource@ujamaa.io
