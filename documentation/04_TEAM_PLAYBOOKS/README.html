📄🎯 Development Methodology

How we build software at Ujamaa.

Key Principles

  • ✅ Agile - 2-week sprints
  • ✅ Test-Driven - Tests first
  • ✅ Code Review - All changes reviewed
  • ✅ Continuous Integration - Automated testing
  • ✅ Documentation - Docs with code

Sprint Cycle

Week 1: Sprint Planning → Development → Code Review Week 2: Development → Testing → Sprint Review → Retro

Quality Gates

  • ESLint passing
  • Tests passing (>90% coverage)
  • Code review approved
  • Documentation updated

📄🔧 Git/GitHub Workflow

Version control best practices.

Branch Strategy

main (protected) └─→ develop (integration) └─→ feature/* (feature branches) └─→ bugfix/* (bug fixes) └─→ hotfix/* (urgent fixes)

Commit Message Convention

type(scope): subject body (optional) footer (optional)

Types:
- feat: New feature
- fix: Bug fix
- docs: Documentation
- style: Formatting
- refactor: Refactoring
- test: Tests
- chore: Maintenance

Pull Request Process

  1. Create branch from develop
  2. Make changes with tests
  3. Run linting & tests locally
  4. Create PR with clear description
  5. Code review (1+ approvals required)
  6. Merge to develop
  7. Delete branch after merge
