🏢 Phase 1: Onboarding & KYB Verification

1

Register as Industrial Operator

What happens: You create an account and select your business type:

  • 🏭 Manufacturing — Factories, production facilities (€100K-€5M, 10-12% APY)
  • 🌱 Agricultural — Farms, cooperatives (€50K-€2M, 12-15% APY)
  • 📦 Trading — Import/export, trade finance (€50K-€3M, 8-10% APY)
Database records created:
  • User record with role INDUSTRIAL_OPERATOR
  • InvestorProfile with your company details and wallet address
  • Document records for KYB (incorporation cert, tax registration, UBO declaration)

2

Submit Asset for Certification

What happens: You describe the asset you want to use as collateral:

📝 Asset Types Available:

Asset Type Example Best For

📦 INVENTORY 1,000 cotton bales in warehouse Manufacturers, farmers
🏭 PRODUCTION Monthly output of 500 tons Factories

  • Asset type: Invoice, Inventory, Production, etc.
  • Value: Asset value in EUROD (e.g., €500,000)

  • Warehouse location: Where the asset is stored
  • Description: Detailed description of the asset
  • Supporting documents: Invoices, warehouse receipts, contracts
🔑 Key Point: At this stage, you're not yet borrowing money. You're simply declaring that you have an asset, and requesting certification from GDIZ/SIPI (the physical verification authority).

GDIZ/SIPI Certifies Your Asset

What happens: A certified verifier (GDIZ/SIPI) physically inspects your asset and confirms it exists:

  • They visit your warehouse/facility
  • They verify quantity, quality, and ownership
  • They upload verification documents to IPFS
  • They call IndustrialGateway.certifyAsset() on-chain
On-chain action (Smart Contract):

🎫 Phase 2: uGT ERC-3643NFT Minting

4

Guarantee Token (uGT) is Minted

What happens: Once your asset is certified, the system mints a Guarantee Token (uGT) — an ERC-3643NFT (ERC-721 + ERC-3643 compliance) that represents your collateral:

🎫 What is a uGT?

  • ERC-3643NFT: Non-fungible token with ERC-3643 compliance (identity verification + transfer restrictions)
  • Collateral: It serves as security for the financing
  • Metadata: Contains asset description, warehouse location, certificate ID
  • Compliance: Only verified entities can hold or transfer uGT
  • Restrictions: Can only be transferred between the pool, industrial, or auction winner

🔄 How It Works:

  1. Pool manager creates financing: LiquidityPool.createFinancing(...)
    • Selects pool family (e.g., POOL_INDUSTRIE)
    • Sets amount (€500,000), interest rate (12%), duration (365 days)
    • References your certificate ID
  2. uGT is minted automatically: The contract calls GuaranteeTokenizer.mintGuarantee()
    • Mints ERC-3643NFT to your wallet address
    • Links it to your certificate
    • Assigns it to the pool as collateral
// GuaranteeTokenizer.sol function mintGuarantee( address industrial, uint256 certificateId, uint256 value, uint256 expiryDate, bytes32 stockHash, string description, string warehouseLocation ) returns (uint256 tokenId) { _safeMint(industrial, tokenId); // Stores Guarantee struct with all metadata // tokenId is now your collateral ERC-3643NFT }
📦 uGT Lifecycle:
  1. Minted: When financing is created
  2. Held by pool: As collateral while loan is active
  3. Returned: When you fully repay the loan
  4. OR Liquidated: If you default (sold at auction)

💰 Phase 3: Funds Deployment

5

Pool Deploys Funds to You

What happens: The pool manager approves the financing and transfers EUROD to your wallet:

  1. Validation:
    • ✅ Your uGT collateral is minted and linked to financing
    • ✅ Pool has sufficient available funds
    • ✅ Diversification limits not exceeded (max 20% per industrial, max 40% per asset class)
  2. Funds Transfer:
    // On-chain: LiquidityPool.deployFunds(financingId, amount) function deployFunds(uint256 financingId, uint256 amount) external { // Transfer EUROD from pool to industrial ujeurToken.safeTransfer(financing.industrial, amount); emit FundsDeployed(financingId, amount); }
✅ You've received the funds! The EUROD is now in your wallet. You can use it for operations, equipment purchases, inventory, etc. The uGT ERC-3643NFT remains as collateral until you repay.
⚠️ Important: The financing has a defined term (e.g., 365 days). You must repay the principal + interest by the maturity date. If you don't, the financing is marked as defaulted and your uGT collateral may be liquidated.

💳 Phase 4: Repayment

6

Repay the Financing

What happens: Over time, you make repayments to the pool. Each repayment is recorded on-chain:

📝 Repayment Flow:

  1. Initiate Repayment: You transfer EUROD to the pool (or backend records the repayment)
  2. Record On-Chain: Pool manager calls LiquidityPool.recordRepayment(financingId, amount)
  3. Tracking Updates:
    • financing.amountRepaid += amount
    • Check if fully repaid: amountRepaid >= principal + interest
  4. Yield Distribution: If your repayment includes interest, the yield portion is added to the ULPTokenizer pool, increasing NAV for all investors
// LiquidityPool.sol function recordRepayment(uint256 financingId, uint256 amount) returns (bool) { Financing storage financing = financings[financingId]; financing.amountRepaid += amount; // Check if fully repaid uint256 totalOwed = financing.principal + (financing.principal * interestRate / 10000); bool isFullyRepaid = financing.amountRepaid >= totalOwed; if (isFullyRepaid) { financing.isRepaid = true; } // Add yield to uLP pool uint256 yieldPortion = amount > financing.principal ? amount - financing.principal : 0; if (yieldPortion > 0) { ulpToken.addYield(yieldPortion, "Financing #123"); } return isFullyRepaid; }

7

uGT Returned to You

What happens: Once you've fully repaid the financing, your collateral is returned:

  1. Pool manager calls: GuaranteeTokenizer.redeemGuarantee(tokenId)
  2. ERC-3643NFT transferred back: The uGT ERC-3643NFT is transferred from the pool back to your wallet
  3. Marked as redeemed: guarantee.isRedeemed = true
🎉 Congratulations! You've fully repaid the financing. Your uGT collateral has been returned. You can now:
  • Apply for new financing using the same collateral
  • Use the uGT as proof of your successful repayment history
  • Build your reputation score for better rates in the future

🔥 Phase 5: uGT Burning (Alternative to Return)

8

When uGT Gets Burned Instead of Returned

What happens: In certain scenarios, the uGT collateral is burned rather than returned:

🔥 Scenarios Where uGT is Burned:

  1. Default & Liquidation: If you fail to repay the financing
    • Pool manager calls CollateralManager.liquidateCollateral(tokenId, auctionWinner)
    • The uGT is transferred to the auction winner (who pays the outstanding debt)
    • After auction, the uGT may be burned if the collateral is forfeited
  2. Asset Depreciation: If the underlying asset loses significant value
    • CollateralManager detects LTV ratio breach (loanValue / collateralValue > maxLTV)
    • Margin call issued; if not met, collateral is liquidated
  3. Regulatory Action: If compliance issues arise
    • Compliance Officer flags the collateral
    • Regulatory authority orders asset freeze or liquidation
// CollateralManager.sol - Liquidation Flow function liquidateCollateral(uint256 tokenId, address liquidator) external onlyRole(POOL_MANAGER_ROLE) { require(collateralState[tokenId] == CollateralState.LOCKED, "Collateral not locked"); // Transfer collateral to liquidator guaranteeTokenizer.safeTransferFrom(address(this), liquidator, tokenId); // Update state collateralState[tokenId] = CollateralState.LIQUIDATED; liquidationDetails[tokenId] = Liquidation({ liquidator: liquidator, timestamp: block.timestamp, recoveryAmount: _calculateRecoveryAmount(tokenId) }); emit CollateralLiquidated(tokenId, liquidator, block.timestamp); }
⚠️ Important: If your uGT is liquidated, your reputation score is significantly impacted. This affects your ability to secure future financing and may result in higher interest rates.

📊 CollateralManager Integration (NEW April 11, 2026)

🔐 What is CollateralManager?

The CollateralManager contract (0xCeA194CbdEcff05221C009Fdf9d7e373111022A5) is the centralized risk management layer for all uGT collateral:

📋 Key Functions:

Function Purpose When Called
lockCollateral(tokenId, financingId) Locks uGT as collateral for a specific financing When financing is created
releaseCollateral(tokenId) Releases uGT back to operator after full repayment When financing is fully repaid
liquidateCollateral(tokenId, liquidator) Transfers uGT to liquidator on default On default or margin call breach
getCollateralRatio(tokenId) Returns current LTV ratio Anytime (view function)
isCollateralHealthy(tokenId) Checks if LTV is within acceptable range Monitoring dashboards
🎯 LTV Calculation:
LTV Ratio = (Loan Value ÷ Collateral Value) × 100%

Example: If you borrow €500K against inventory worth €750K:
LTV = (500,000 ÷ 750,000) × 100% = 66.67%

Max LTV Threshold: 75% (configurable per asset class)
If LTV exceeds 75%, a margin call is triggered.
// CollateralManager.sol - LTV Validation function validateLTV(uint256 tokenId, uint256 loanAmount) public view returns (bool) { uint256 collateralValue = _getCollateralValue(tokenId); uint256 currentLTV = (loanAmount * 10000) / collateralValue; // Basis points uint256 maxLTV = assetClassMaxLTV[_getAssetClass(tokenId)]; return currentLTV <= maxLTV; } // Margin call check function checkMarginCall(uint256 tokenId) public view returns (bool marginCallRequired) { uint256 collateralValue = _getCollateralValue(tokenId); uint256 outstandingDebt = _getOutstandingDebt(tokenId); uint256 currentLTV = (outstandingDebt * 10000) / collateralValue; uint256 marginCallThreshold = 7000; // 70% in basis points return currentLTV > marginCallThreshold; }
