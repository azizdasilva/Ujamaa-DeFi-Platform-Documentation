1. Overview

The Credit Limit API provides endpoints for viewing, calculating, and managing user credit limits. Credit limits determine the maximum borrowing capacity for industrial operators on the Ujamaa DeFi Platform.

Key Concepts:

  • Credit Limit: Maximum amount (in EUR/EUROD) a user can borrow at any given time
  • Available Credit: credit_limit - outstanding_loan_principal
  • Verification Gate: Credit limits are only active when both KYC and KYB are APPROVED
  • Admin Override: Administrators can manually set credit limits, overriding automatic calculation

3. GET /me

Endpoint: GET /api/v2/credit-limits/me

Returns the current credit limit and available credit for the authenticated user.

4. POST /me/recalculate

Endpoint: POST /api/v2/credit-limits/me/recalculate

Forces a recalculation of the current user's credit limit. Useful after asset certification or KYB upgrade.

No request body required.

Response: Same format as GET /me.

6. PUT /users/:id (Admin)

Endpoint: PUT /api/v2/credit-limits/users/{user_id}

Manually override a user's credit limit. Takes precedence over automatic calculation.

Request

PUT /api/v2/credit-limits/users/5 HTTP/1.1
Authorization: Bearer admin-token-mvp
Content-Type: application/json

{
  "credit_limit": 150000,
  "reason": "Strategic partner — Green Cotton expansion phase"
}

Response (200 OK)

{
  "success": true,
  "message": "Credit limit overridden to 150,000.00 EUR",
  "user_id": 5,
  "credit_limit": 150000.00,
  "available_credit": 150000.00,
  "method": "ADMIN_OVERRIDE",
  "reason": "Strategic partner — Green Cotton expansion phase"
}

7. POST /users/:id/reset (Admin)

Endpoint: POST /api/v2/credit-limits/users/{user_id}/reset

Remove admin override and reset to auto-calculated value.

Request

POST /api/v2/credit-limits/users/5/reset HTTP/1.1
Authorization: Bearer admin-token-mvp

Response (200 OK)

{
  "success": true,
  "message": "Credit limit reset to auto-calculated value",
  "new_credit_limit": 185000.00,
  "method": "COLLATERAL_BASED"
}

8. Credit Limit Calculation Formula

The upgraded system uses a multi-factor pipeline benchmarked against Aave V3, Morpho Blue, and Basel III:

Step 1: Verification Gate → KYB must be APPROVED
Step 2: Risk-Weighted Collateral → Σ(Asset_value × Asset_class_factor)
Step 3: 5Cs Risk Score → Weighted composite (0-100)
Step 4: Tier Multiplier → Based on score (AAA=1.0, A=0.8, etc.)
Step 5: Concentration → ±adjustment for diversification
Step 6: Reputation → Compounding multiplier from repayment history
Step 7: Global Cap → min(result, €500,000)

Final = RWC × Tier_Multiplier × Concentration × Reputation

Risk-Weighted Collateral Factors

Asset ClassFactorRationale
Receivables75%High liquidity, verifiable
Inventory55%Moderate liquidity, storage risk
Equipment45%Depreciation, illiquid
Real Estate65%Stable value, illiquid
Contracts/Other30-35%Counterparty risk

Tier Ratings

ScoreRatingMultiplierMax LTV

Reputation System

On-time repayment: +2% per loan (compounding, max +20%). Early repayment: +5% for 12 months. Late repayment: -5% for 6 months. Default: -50% for 24 months.
