📝 Documentation Style Guide

Guidelines for writing clear, consistent documentation

📖 Document Info

Status: ✅ Complete | Audience: All Contributors

📑 Writing Principles

Core Principles
  • Write for your audience
  • Be clear and concise
  • Use active voice
  • Include examples
  • Keep it up to date

📝 Formatting Standards

ElementFormat
HeadingsH1 for title, H2 for sections, H3 for subsections
CodeInline: code, Blocks: triple backticks
Links[text](url)
Emphasis*italic*, **bold**
ListsBullets (-) or numbered (1.)

📋 Document Structure

  1. Title and subtitle
  2. Document info callout
  3. Overview section
  4. Main content sections
  5. Related documents
  6. Footer with metadata

🎨 Callout Usage

TypeUse For
Info (blue)General information
Success (green)Positive information, tips
Warning (yellow)Cautions, important notes
Error (red)Critical warnings, errors

📊 Tables

  • Use for structured data
  • Keep columns to 3-4 maximum
  • Use clear headers
  • Keep content concise

🔗 Cross-References

  • Link to related documents
  • Use descriptive link text
  • Include "Related" section at bottom
  • Use relative paths

✅ Review Checklist

Before Publishing
  • ✓ Clear and accurate
  • ✓ Proper formatting
  • ✓ Links working
  • ✓ No typos
  • ✓ Examples included
  • ✓ Related docs linked
