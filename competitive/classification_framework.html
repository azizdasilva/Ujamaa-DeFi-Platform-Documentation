📐 Competitor Classification Framework

Ujamaa DeFi Platform — Competitive Intelligence System

📅 Document Info

Version: 1.0 | Date: March 27, 2026 | Purpose: Standardized methodology for identifying, scoring, and classifying competitors

1️⃣1. Overview

This framework provides a systematic approach to evaluate business similarity between Ujamaa DeFi Platform and any identified competitor. The framework uses an 8-criteria weighted scoring grid to compute a Weighted Similarity Score (WSS) out of 100.

Competitors are classified into three classes based on their WSS:

  • Class 1 — Direct Competitors (WSS: 65–100)
  • Class 2 — Indirect Competitors (WSS: 35–64)
  • Class 3 — Peripheral Competitors (WSS: 0–34)

2️⃣2. Scoring Grid

2.1 Criteria & Weights

# Criterion Weight Description
1 Target Market Geography 20% Same continent / region / country focus
2 Target User Segment 15% Unbanked, underbanked, retail, SME, diaspora, institution
3 Core Product Overlap 20% Number of overlapping DeFi features
4 Blockchain / Tech Stack Similarity 10% Same chain, same standards, similar architecture
5 Business Model Similarity 10% Revenue model, fee structure, token utility
6 Social / Inclusion Mandate 10% Explicit financial inclusion or community-ownership mission
7 Token Economy Overlap 8% Similar tokenomics design, staking, governance
8 Partnership Ecosystem Overlap 7% Shared partners, protocols, or rails
Total 100%

2.2 Weight Rationale

Weights reflect Ujamaa DeFi Platform's strategic positioning:

  • Geography (20%): African focus is core to Ujamaa DeFi Platform's identity and market strategy
  • Product Overlap (20%): Direct feature competition is a primary threat indicator
  • User Segment (15%): Serving the same customers creates direct competition for adoption
  • Social Mandate (10%): Financial inclusion mission differentiates Ujamaa from pure-profit players
  • Business Model (10%): Similar revenue models indicate comparable go-to-market approaches
  • Tech Stack (10%): Technology choices affect integration partnerships and developer talent competition
  • Token Economy (8%): Token design affects investor attraction and retention
  • Partnerships (7%): Overlapping partners create resource competition

3️⃣3. Scoring Rubric (0–5 Scale)

Criterion 1: Target Market Geography (20%)

Score Descriptor Criteria
0 No Overlap No presence in Africa; focused on other continents
1 Minimal Overlap Incidental African presence (<5% of users/volume)
2 Low Overlap Some African presence (5-15% of users/volume); not strategic focus
3 Moderate Overlap Meaningful African presence (15-30%); expanding in region
4 High Overlap Significant African focus (30-60%); multiple country operations
5 Identical Primary focus on Africa (60%+); OHADA/West Africa concentration

Ujamaa Benchmark: 100% African focus (OHADA zone primary)

Criterion 2: Target User Segment (15%)

Score Descriptor Criteria
0 No Overlap Completely different segments (e.g., crypto traders vs. SME borrowers)
1 Minimal Overlap Adjacent segments with little crossover (e.g., retail crypto users)
2 Low Overlap Some segment overlap (e.g., retail + institutional mix)
3 Moderate Overlap Significant overlap (e.g., institutional investors + SMEs)
4 High Overlap Strong overlap (e.g., institutional credit + emerging market SMEs)
5 Identical Same segments: Institutional investors (€100K+) + African SME borrowers

Ujamaa Benchmark: Institutional investors (€100K+ minimum) + African SMEs (€500K-€5M financing)

Criterion 3: Core Product Overlap (20%)

Score Descriptor Criteria
0 No Overlap No shared product features
1 Minimal Overlap 1 overlapping feature (e.g., stablecoin support only)
2 Low Overlap 2 overlapping features (e.g., lending + stablecoin)
3 Moderate Overlap 3-4 overlapping features (e.g., lending + pools + compliance)
4 High Overlap 5-6 overlapping features (e.g., lending + pools + compliance + tokenization)
5 Identical 7+ overlapping features: Liquidity pools, yield-bearing tokens, RWA tokenization, compliance/KYC, institutional custody, bank integration, African focus

  1. Liquidity pools
  2. Yield-bearing tokens (uLP)
  3. RWA tokenization (SME financing)
  4. Collateral NFTs (Guarantee Token)
  5. ERC-3643 compliance
  6. Institutional custody (Fireblocks)
  7. Bank escrow integration
  8. African SME focus

Criterion 4: Blockchain / Tech Stack Similarity (10%)

Score Descriptor Criteria
0 No Overlap Different blockchain paradigm (e.g., non-blockchain fintech)
1 Minimal Overlap Different chains, no shared standards (e.g., Solana vs. Ethereum)
2 Low Overlap Some shared infrastructure (e.g., EVM-compatible but different L1)
3 Moderate Overlap Same L1/L2 (e.g., Polygon) but different token standards
4 High Overlap Same chains + similar standards (e.g., Polygon + ERC-20/ERC-721)
5 Identical Same stack: Polygon/Ethereum + ERC-3643 + similar infrastructure (Fireblocks, Chainlink, etc.)

Ujamaa Stack: Polygon (primary), Ethereum (settlement), ERC-3643, Fireblocks, Chainlink

Criterion 5: Business Model Similarity (10%)

Score Descriptor Criteria
0 No Overlap Completely different model (e.g., donation-based, grant-funded)
1 Minimal Overlap Different revenue logic (e.g., trading fees vs. lending yield)
2 Low Overlap Some similarity (e.g., fee-based but different fee structure)
3 Moderate Overlap Similar model (e.g., management + performance fees)
4 High Overlap Very similar (e.g., 2% mgmt + 20% performance + origination fees)
5 Identical Same model: 2% management fee + 20% performance fee (8% hurdle) + 1-2% origination

2 Low Overlap Some inclusion rhetoric; limited concrete programs
3 Moderate Overlap Clear inclusion mission; measurable targets (e.g., financial access)

3 Moderate Overlap Yield-bearing token with different mechanics
4 High Overlap Yield-bearing security token with compliance (e.g., ERC-3643)

5 Identical Same key partners: Fireblocks, Chainlink, African industrial networks, bank escrow

4️⃣4. WSS Calculation Formula

🧮 Formula
Weighted Similarity Score (WSS) = Σ(Criterion Score × Weight)

Where:
- Criterion Score: 0-5 (from rubric above)
- Weight: Percentage from Section 2.1

Maximum WSS = 5 × 100% = 100 points

Calculation Example

Criterion Score Weight Weighted Score
Geography 5 20% 1.00
User Segment 4 15% 0.60
Product Overlap 5 20% 1.00
Tech Stack 4 10% 0.40
Business Model 4 10% 0.40
Social Mandate 5 10% 0.50
Token Economy 5 8% 0.40
Partnerships 3 7% 0.21
Total 100% 4.51 × 20 = 90.2

WSS = 90.2 → Class 1 (Direct Competitor)

5️⃣5. Classification Classes

Class 1 — Direct Competitors (WSS: 65–100)

⚠️ Definition

Closest rivals serving the same users, in the same geographies, with overlapping products and similar mission.

Characteristics:

  • Compete for the same institutional investors (€100K+)
  • Target African SMEs or emerging market borrowers
  • Offer liquidity pools or RWA tokenization
  • Have regulatory compliance framework
  • Explicit financial inclusion or development mission

Monitoring Cadence: Monthly reviews
Strategic Response: Priority competitive countermeasures

Class 2 — Indirect Competitors (WSS: 35–64)

⚠️ Definition

Significant overlap in some dimensions (e.g., same geography but different product, or same product but different geography).

Characteristics:

  • May serve African market but with different products (e.g., payments vs. lending)
  • May offer similar products but in different geographies (e.g., RWA in US/EU)
  • Could become Class 1 through geographic expansion or product pivots
  • Potential partnership opportunities

Monitoring Cadence: Quarterly reviews
Strategic Response: Monitor for convergence; evaluate partnership potential

Class 3 — Peripheral Competitors (WSS: 0–34)

ℹ️ Definition

Distant players in broader DeFi or African fintech space with limited direct overlap.

Characteristics:

  • Global DeFi protocols with minimal Africa presence
  • African fintechs without blockchain/DeFi features
  • Pure crypto exchanges without lending/tokenization
  • Different user segments (retail traders vs. institutional investors)

Monitoring Cadence: Quarterly reviews (alongside Class 2)
Strategic Response: Monitor for disruptive moves; low priority

6️⃣6. Scoring Methodology

6.1 Evidence Requirements

Every score must be justified with specific facts from research:

Evidence Type Examples
Public Statements Website copy, whitepapers, investor presentations
Product Documentation Feature lists, API docs, user guides
Financial Data TVL, AUM, funding rounds, revenue disclosures
Partnership Announcements Press releases, blog posts, integration docs
Regulatory Filings License applications, compliance disclosures
Social Media Twitter/X, LinkedIn, community channels
Third-Party Reports CoinMarketCap, DeFiLlama, Crunchbase, news articles

6.2 Handling Data Gaps

If data is unavailable for a criterion:

  1. Note the gap explicitly in the competitor profile
  2. Use conservative scoring (assume lower similarity unless proven otherwise)
  3. Flag for follow-up research in monitoring plan

6.3 Score Validation

Scores should be:

  • Replicable: Another analyst using the same evidence should arrive at similar scores
  • Documented: All scores include citations to source evidence
  • Reviewed: Class 1 scores require peer review before finalization

7️⃣7. Dynamic Reclassification

Competitors may move between classes over time. Triggers for reclassification:

Trigger Example Action
Geographic Expansion Global RWA platform launches Africa desk Reclassify upward
Product Launch African payments app adds lending pools Reclassify upward
Funding Round Competitor raises €50M+ for expansion Increase monitoring
Partnership Announcement Competitor partners with GDIZ-equivalent Recalculate partnership score
Regulatory Approval Competitor obtains Mauritius FSC license Reclassify upward
Market Exit Competitor shuts down Africa operations Reclassify downward

Review Cycle:

  • Class 1: Monthly
  • Class 2: Quarterly
  • Class 3: Quarterly (lighter review)

8️⃣8. Limitations & Assumptions

8.1 Known Limitations

  1. Data Availability: Private companies may not disclose TVL, users, or revenue
  2. Self-Reporting Bias: Marketing materials may overstate Africa focus or impact
  3. Rapid Change: DeFi landscape evolves quickly; scores may become outdated
  4. Subjectivity: Some criteria (e.g., social mandate) involve judgment calls

8.2 Key Assumptions

  1. Africa Focus Premium: Geography weighted heavily (20%) due to Ujamaa DeFi Platform's core positioning
  2. Institutional Threshold: €100K minimum defines institutional segment
  3. Compliance Importance: ERC-3643 or equivalent compliance is a key differentiator
  4. Impact Authenticity: Stated mission must be backed by measurable targets
