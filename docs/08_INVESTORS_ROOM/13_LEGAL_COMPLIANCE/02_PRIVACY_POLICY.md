# 4. DATA RETENTION

## 4.1 Retention Periods

| Data Type | Retention Period | Reason |
|-----------|-----------------|--------|
| **Account Data** | Duration of relationship + 5 years | Regulatory requirement |
| **KYC/AML Data** | 5 years after relationship ends | AML legal requirement |
| **Transaction Data** | 7 years | Tax/legal requirement |
| **Communications** | 3 years after last contact | Business need |
| **Usage Data** | 2 years | Analytics |
| **Blockchain Data** | Permanent (immutable) | Blockchain nature |

## 4.2 Deletion

Upon request or account closure:
- Personal data is deleted where legally permitted
- Some data must be retained for compliance
- Blockchain data cannot be deleted

## 4.3 Anonymization

Data may be anonymized and retained indefinitely for:
- Statistical analysis
- Service improvement
- Research purposes

# 5. DATA SECURITY

## 5.1 Security Measures

| Category | Measures |
|----------|----------|
| **Encryption** | AES-256 at rest, TLS 1.3 in transit |
| **Access Control** | Role-based access, MFA required |
| **Network Security** | Firewalls, intrusion detection |
| **Application Security** | OWASP compliance, regular testing |
| **Physical Security** | Secure data centers |
| **Monitoring** | 24/7 SOC monitoring, SIEM |

## 5.2 Data Breach Response

In case of data breach:
1. Contain and assess breach
2. Notify affected users (within 72 hours if required)
3. Notify regulators (as required)
4. Remediate vulnerabilities
5. Document and review

## 5.3 Your Responsibilities

You are responsible for:
- Protecting your login credentials
- Securing your private keys
- Using strong passwords
- Enabling two-factor authentication
- Reporting suspicious activity

# 6. YOUR PRIVACY RIGHTS

## 6.1 GDPR Rights (EU/EEA)

| Right | Description |
|-------|-------------|
| **Access** | Request copy of personal data |
| **Rectification** | Correct inaccurate data |
| **Erasure** | Request deletion (where applicable) |
| **Portability** | Receive data in machine-readable format |
| **Restriction** | Request limitation of processing |
| **Objection** | Object to certain processing |
| **Withdraw Consent** | Withdraw consent at any time |

## 6.2 CCPA/CPRA Rights (California)

| Right | Description |
|-------|-------------|
| **Know** | Request disclosure of data practices |
| **Delete** | Request deletion of personal data |
| **Opt-Out** | Opt out of data sale/sharing |
| **Correct** | Request correction of inaccurate data |
| **Limit** | Limit use of sensitive data |
| **Non-Discrimination** | Equal service regardless of privacy choices |

## 6.3 Other Jurisdiction Rights

| Jurisdiction | Rights |
|--------------|--------|
| **Nigeria (NDPR)** | Access, rectification, erasure |
| **Kenya (Data Protection Act)** | Access, correction, deletion |
| **South Africa (POPIA)** | Access, correction, deletion |
| **Ghana (Data Protection Act)** | Access, correction, objection |

## 6.4 Exercising Your Rights

To exercise privacy rights:
1. Submit request to dpo@ujamaa-defi.com
2. Provide identity verification
3. Specify right being exercised
4. Allow 30 days for response
