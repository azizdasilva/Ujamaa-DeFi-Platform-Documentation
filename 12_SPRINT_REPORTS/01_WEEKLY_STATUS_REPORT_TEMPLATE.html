📊 Weekly Status Report Template

Template for weekly development status reporting

📖 Document Info

Status: ✅ Complete | Type: Template

📑 Overview

This template provides the standard format for weekly development status reports. Reports should be submitted every Friday by 5 PM.

📝 Report Template

WEEKLY STATUS REPORT Week [Number] - [Date Range] TEAM MEMBER: [Name] ROLE: [Role] ───────────────────────────── 1. COMPLETED THIS WEEK ───────────────────── • [Task 1] - [Status] • [Task 2] - [Status] • [Task 3] - [Status] 2. IN PROGRESS ──────────── • [Task 1] - [X]% complete • [Task 2] - [X]% complete 3. BLOCKERS/ISSUES ──────────────── • [Issue 1] - [Impact] • [Issue 2] - [Impact] 4. NEXT WEEK PLAN ─────────────── • [Task 1] • [Task 2] • [Task 3] 5. METRICS ──────── Commits: [Number] PRs Merged: [Number] Tests Added: [Number] Bugs Fixed: [Number] 6. NOTES/COMMENTS ─────────────── [Any additional notes]

📋 Section Guidelines

SectionWhat to Include
1. CompletedTasks finished this week with status
2. In ProgressOngoing tasks with completion percentage
3. BlockersIssues preventing progress, with impact
4. Next WeekPlanned tasks for next week
5. MetricsQuantifiable development metrics

⏱️ Submission Schedule

  • Due: Every Friday by 5:00 PM
  • Submit to: Team lead, project channel
  • Format: Markdown or HTML
  • Review: Monday standup

📊 Example Metrics

💻
Commits
Code commits to repository
🔀
PRs Merged
Pull requests merged
✅
Tests Added
New test cases written
🐛
